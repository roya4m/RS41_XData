Configure and run the scripts

Script configuration

The menu "Report Templates and Scripts" (Administration → Report Templates and Scripts → Scripts) allows script configurations to be added and removed.

Add and remove scripts

Explanation of fields:

  • Name: This is the script name which is used as a folder name to store the main script and its include files. Changing the name will result in renaming of the folder as well (usually under C:\ProgramData\MW41\Scripts).
  • Active: Only active scripts will be started at the beginning of a sounding or at system startup. None, unselecting this checkbox will not stop the script.
  • Run at system startup
    • selected: the script will be started at system start up (if selected a system restart is needed to start the script)
    • not selected: the script will be started at the beginning of the sounding.

When selecting the script name link (here "MainCustomer") the dialog "Manage script group" will be displayed:

Manage Script Group

This dialog allows to upload the script and its include files. In addition, comand-line argumens can be given to the main script.

Start and stop the scripts

Start script

The script is started automatically if configured active (see Script configuration).

You can manually start a script from command line with ScriptExecutor.exe by giving script file as a parameter. ScriptExecutor.exe should be run in [installation directory]\Bin directory. In this case, make sure that the script is not configured to start automatically.

Stopping the script

Scripts should stop them selfs by calling SoundingInterface.StopScript().

If "Run at startup" option is not set, system will kill automatically started scripts if those are running when new sounding is created.
